What is FinderConnect?

FinderConnect is a powerful location finder widget that helps patients discover and connect with healthcare providers in their area. The widget seamlessly integrates into your website and provides an interactive map experience with partner branding and full marketing attribution support.

Interactive Map Smart Filtering Marketing Attribution Mobile Responsive

Marketing Attribution & UTM Parameters

FinderConnect automatically captures and preserves marketing attribution data throughout the patient journey. UTM parameters and click IDs from your campaigns are automatically passed through to the embedded iframe, ensuring accurate tracking from initial discovery to appointment booking.

Supported Attribution Parameters:
  • utm_source - Identifies the source of traffic (e.g., google, facebook)
  • utm_medium - Identifies the medium (e.g., cpc, email, social)
  • utm_campaign - Identifies the specific campaign
  • utm_term - Identifies paid search keywords
  • utm_content - Identifies specific ad or link variations
  • gclid - Google Click ID for Google Ads tracking
  • fbclid - Facebook Click ID for Facebook Ads tracking
  • msclkid - Microsoft Click ID for Bing Ads tracking

Quick Start Integration

Add the FinderConnect SDK script to your HTML page before the closing </body> tag:

<!-- FinderConnect SDK --> <script src="https://app.dev.intelibly.com/finderconnect.js?partner_id=YOUR_PARTNER_ID"></script> <!-- Add the widget element where you want it to appear --> <finderconnect></finderconnect>

Advanced Configuration

Customize the widget appearance and behavior with attributes:

<finderconnect partner-id="YOUR_PARTNER_ID" width="100%" height="600px" primary-color="#06A3DA" secondary-color="#13547A" ></finderconnect>

Configuration Attributes

Attribute Description Required Default
data-partner-id Your unique partner identifier Yes -
data-primary-color Primary brand color (hex format) No #06A3DA
data-secondary-color Secondary brand color (hex format) No #13547A
data-default-location Initial map center location No User's location
data-radius Search radius in miles No 10
data-show-filters Show/hide filter options No true

Single Instance Requirement

The FinderConnect SDK is designed to render only one widget instance per page. If multiple <finderconnect> elements are detected, the SDK will:

  • Initialize only the first instance found in the DOM
  • Display a console warning: "Multiple <finderconnect> elements detected. Only the first instance will be initialized."
  • Add a data-fc-ignored="true" attribute to subsequent elements
Best Practice: Always include exactly one <finderconnect> element per page to avoid confusion and ensure proper initialization.
